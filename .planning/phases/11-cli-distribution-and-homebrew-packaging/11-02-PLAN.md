  port: number;
  host?: string;
  staticDir?: string | null;
  logger?: boolean;

Create `server/src/cli.ts` exporting these testable helpers and using them in the CLI entry:
- `parseCliArgs(argv: string[])`
- `resolveRootDir({ envRootDir, rootArg, cwd })`
- `resolvePort({ requestedPort, defaultPort })`
- `maybeOpenBrowser({ url, openBrowser })`
- `main(argv = process.argv.slice(2))`

Use these exact runtime rules:
- default port is `4310`
- probe range is `4310..4320`
- explicit `--port` collision throws `Port <n> is already in use`
- `ROOT_DIR` beats positional arg; positional arg beats cwd
- `--no-open` disables browser open; default is open

Update `server/src/index.ts` so repo-local startup also goes through `startMarkyServer` instead of duplicating bootstrap logic.
