  <action>
    Create `client/src/components/EditorPane.tsx`.

    **Props interface:**
    ```typescript
    interface EditorPaneProps {
      tab: Tab;
      dispatch: React.Dispatch<TabAction>;
      onLinkClick: (path: string) => void;
    }
    ```

    **State:** Local `editContent: string` — initialized from `tab.content ?? ''` when edit mode opens. This is the single source of truth while editing; the reducer's `tab.content` is only updated by SET_CONTENT (initial load) and is not written on every keystroke (prevents cursor-jumping pitfall from RESEARCH.md).

    **useEffect for editContent initialization:** When `tab.editMode` becomes true and `tab.content` is not null, set `editContent` to `tab.content`. Use `tab.id` as the dep to reset when tab switches.

    **handleChange:** Called on every CodeMirror onChange. Updates local `editContent` state. On first call (when `!tab.dirty`), dispatches `SET_DIRTY`. Does NOT dispatch SET_CONTENT on every keystroke.

    **onSaved callback (stable, useCallback):** Dispatches `CLEAR_DIRTY` for `tab.id`. Does NOT update editContent — the editor already has the latest content.

    **useAutoSave call:** `useAutoSave(tab.path, editContent, onSaved)` — only runs when editMode is true (conditionally render or guard with editMode check).

    **Render structure when loading:** Show spinner (same as App.tsx currently).

    **Render structure when not loading and not editMode:** Show MarkdownPreview from `tab.content ?? ''`.

    **Render structure when editMode is true:** Use react-resizable-panels v4:
    ```tsx
    <Group orientation="vertical" className="h-full">
      <Panel id={`preview-${tab.id}`} defaultSize="50%">
        <div className="h-full overflow-y-auto">
          <div className="prose prose-orange max-w-none p-8">
            <MarkdownPreview content={editContent} onLinkClick={onLinkClick} />
          </div>
        </div>
      </Panel>
      <Separator className="h-1 bg-gray-200 hover:bg-orange-400 transition-colors cursor-row-resize" />
      <Panel id={`editor-${tab.id}`} defaultSize="50%">
        <div className="h-full overflow-hidden border-t border-gray-200">
          <MarkdownEditor value={editContent} onChange={handleChange} tabId={tab.id} />
        </div>
      </Panel>
    </Group>
    ```

    **Edit button:** Render a small "Edit" / "Preview" toggle button at the top-right of the pane. When clicked, dispatches `TOGGLE_EDIT` for `tab.id`. Position: absolute top-right inside the pane container, or as a fixed toolbar above the content. Use a simple `<button>` with Tailwind classes matching the app's orange accent style (`text-orange-600 hover:text-orange-800 text-xs font-medium`). Place it in the pane header/toolbar row, consistent with the tab bar above.

    Example button placement — add a toolbar row above the content:
    ```tsx
    <div className="flex justify-end px-4 py-1 border-b border-gray-100 bg-white/40 shrink-0">
      <button
        onClick={() => dispatch({ type: 'TOGGLE_EDIT', id: tab.id })}
        className="text-xs font-medium text-orange-600 hover:text-orange-800 transition-colors"

      </button>
    </div>
    ```
