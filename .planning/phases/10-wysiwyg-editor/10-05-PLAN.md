Rewrite `client/src/__tests__/markdown-roundtrip.test.ts` with real round-trip assertions.

The test approach: Create a TipTap editor instance in the test (headless, no DOM rendering needed), load markdown content via `setContent(md, false, { contentType: 'markdown' })`, then call `editor.getMarkdown()` and assert the output matches the input (allowing for minor whitespace normalization).

```typescript
import { describe, it, expect, beforeAll, afterAll } from 'vitest';
import { Editor } from '@tiptap/core';
import StarterKit from '@tiptap/starter-kit';
import Markdown from '@tiptap/markdown';
import Link from '@tiptap/extension-link';
import Image from '@tiptap/extension-image';
import Table from '@tiptap/extension-table';
import TableRow from '@tiptap/extension-table-row';
import TableCell from '@tiptap/extension-table-cell';
import TableHeader from '@tiptap/extension-table-header';

function roundTrip(md: string): string {
  const editor = new Editor({
    extensions: [
      StarterKit,
      Markdown.configure({ markedOptions: { gfm: true } }),
      Link, Image, Table, TableRow, TableCell, TableHeader,
    ],
    content: md,
    contentType: 'markdown',
  });
  const result = editor.getMarkdown();
  editor.destroy();
  return result;
}

// Helper: normalize whitespace for comparison (trim, collapse multiple newlines)
function normalize(s: string): string {
  return s.trim().replace(/\n{3,}/g, '\n\n');
}
```

For each test, call `roundTrip(input)` and check that the normalized output contains the essential markdown constructs from the input. Use `expect(normalize(output)).toContain(...)` rather than strict equality, since TipTap may add/remove trailing newlines.

**If any round-trip test fails**, investigate:
1. Check if the corresponding TipTap extension is registered
2. Check if `@tiptap/markdown` needs configuration for that construct
3. For frontmatter: may need a custom TipTap node that preserves raw YAML text. If so, create a minimal `FrontmatterNode` extension in `client/src/extensions/frontmatter.ts` and register it in WysiwygEditor.

**For YAML frontmatter specifically:** The `@tiptap/markdown` extension likely does not handle YAML frontmatter natively. Options:
- Create a custom TipTap node that stores the raw frontmatter YAML and renders it as a non-editable gray block
- If round-trip test passes without custom node (MarkedJS preserves unknown blocks), no custom node needed
