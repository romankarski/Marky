## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, bubble toolbar button internal padding |
| sm | 8px | Compact element spacing, slash menu item padding |
| md | 16px | Default element spacing, editor block vertical margins |
| lg | 24px | Section padding, editor content horizontal padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Bubble toolbar offset from selection — 8px above selection bounding box. Slash command dropdown offset — 4px below cursor line.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 13px | 400 (regular) | 1.4 |
| Heading (editor H1) | 28px | 600 (semibold) | 1.2 |
| Heading (editor H2) | 22px | 600 (semibold) | 1.2 |

Note: Editor content inherits the existing `prose prose-orange` typographic scale from Tailwind Typography plugin. The sizes above govern the WYSIWYG surface and its chrome (toolbar labels, slash menu items). Sub-heading levels (H3+) are fully governed by the inherited prose scale and are not declared here.

Accent reserved for:
- Active/selected slash command menu item background (orange-100) and text (orange-600)

- `</>` toggle button active state (when in raw mode)

## Focal Point

Primary visual anchor: the full-height document surface in the center pane. All chrome (bubble toolbar, slash menu, resize handles) is transient and appears only on interaction.

## Interaction Contracts

### Click-to-Edit
- Document renders as styled prose (identical to current preview appearance)
- Hovering over any block shows a subtle left-border highlight (2px solid orange-200, transition 150ms) to signal editability
- Clicking places a blinking cursor at click position; editing begins immediately
- No mode switch button, no toolbar button required

### Floating Bubble Toolbar
- Appears when user selects 1+ characters of text
- Positioned 8px above the selection bounding box, horizontally centered on the selection
- If selection is near top of viewport, toolbar flips below the selection
- Contains 4 buttons at launch: **Bold** (B icon), **Italic** (I icon), **Link** (chain icon), **Inline Code** (backtick icon)
- Button shows pressed/active state (orange-600 fill) when the selection already has that formatting
- Dismissed when selection is cleared (click away or Escape)
- Standard keyboard shortcuts: Cmd+B (bold), Cmd+I (italic), Cmd+K (link), Cmd+E (inline code)

### Slash Command Menu
- Triggered by typing `/` at the start of a new line (empty paragraph)
- Dropdown appears 4px below the current line, left-aligned with the cursor
- Menu filters as user types after `/` (e.g. `/h` shows only heading commands)
- Dismissed with Escape, clicking outside, or Backspace removing the `/`
- Arrow keys navigate items; Enter selects
- Commands at launch:

| Command | Label | Description | Action |
|---------|-------|-------------|--------|
| `/h1` | Heading 1 | Large heading | Convert paragraph to H1 |
| `/h2` | Heading 2 | Medium heading | Convert paragraph to H2 |
| `/h3` | Heading 3 | Small heading | Convert paragraph to H3 |
| `/table` | Table | Insert a table | Insert 3-column x 3-row table with header row |
| `/image` | Image | Insert an image | Open file picker dialog |
| `/code` | Code Block | Insert code | Insert fenced code block with empty language selector |

### Image Handling
- Drag-and-drop: dragging an image file onto the editor surface shows a blue drop indicator line between blocks
- On drop: image is uploaded to server at `/images/` vault root, markdown reference inserted as `![filename](/images/filename.png)`
- `/image` command: opens native file picker, same upload and insert flow
- Clicking an inserted image selects it and reveals 4 corner resize handles
- Dragging a handle resizes proportionally; on release, width is serialized as `<img src="..." width="N">`
- Minimum image width: 64px; maximum: container width

### Raw Mode Toggle
- `</>` icon button in the EditorPane toolbar bar (same row as export buttons)
- Click toggles between WYSIWYG (TipTap) and raw markdown (CodeMirror)
- Transition: crossfade with 150ms ease-in-out opacity transition
- When switching WYSIWYG -> raw: TipTap content is serialized to markdown string, passed to CodeMirror
- When switching raw -> WYSIWYG: CodeMirror value is parsed into TipTap document
- Auto-save works in both modes (same useAutoSave hook, same debounce)
- Active state: icon turns orange-600 when in raw mode

### Un-representable Markdown
- YAML frontmatter: rendered as a non-editable gray block at the top of the document (bg-gray-100, rounded, monospace, 13px); editable only in raw mode
- Raw HTML blocks: rendered as a non-editable code block with a "Raw HTML" label; editable only in raw mode
- Complex constructs (footnotes, definition lists): passed through as raw text nodes; user directed to raw mode for editing

## Library Recommendation (Claude's Discretion)

| Library | Purpose | Rationale |
|---------|---------|-----------|
| `@tiptap/react` | WYSIWYG editor framework | ProseMirror-based, React-native, excellent markdown serialization via `@tiptap/pm`, extensible with custom nodes for slash commands and image resize. Strong community, actively maintained. |
| `@tiptap/starter-kit` | Bundle of common extensions | Includes bold, italic, code, headings, lists, blockquote, code block, horizontal rule — covers core editing needs |
| `@tiptap/extension-link` | Link editing with bubble menu | Handles Cmd+K, paste-as-link, link editing UI |
| `@tiptap/extension-image` | Image node | Renders images in the document, supports width attribute |
| `@tiptap/extension-table` | Table editing | Insert, add/remove rows and columns |
| `@tiptap/extension-placeholder` | Placeholder text | Shows "Start writing" in empty editor |
| `tiptap-markdown` | Markdown serialization/deserialization | Converts between TipTap document and markdown string for on-disk storage |
