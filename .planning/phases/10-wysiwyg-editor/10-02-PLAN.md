Create `client/src/components/WysiwygEditor.tsx`:

```typescript
interface WysiwygEditorProps {
  content: string;               // markdown string from tab.content
  onChange: (markdown: string) => void;  // feeds into editContent state
  onLinkClick: (path: string) => void;  // wiki-link / .md link click handler
}
```

Implementation details:
1. Use `useEditor` from `@tiptap/react` with these extensions:
   - `StarterKit` from `@tiptap/starter-kit` (bold, italic, strike, code, headings, lists, blockquote, code block, HR, history)
   - `Markdown.configure({ markedOptions: { gfm: true } })` from `@tiptap/markdown`
   - `Link.configure({ openOnClick: false })` from `@tiptap/extension-link`
   - `Image.configure({ inline: false })` from `@tiptap/extension-image`
   - `Table.configure({ resizable: true })` from `@tiptap/extension-table`
   - `TableRow` from `@tiptap/extension-table-row`
   - `TableCell` from `@tiptap/extension-table-cell`
   - `TableHeader` from `@tiptap/extension-table-header`
   - `Placeholder.configure({ placeholder: 'Click anywhere to begin, or type / for commands' })` from `@tiptap/extension-placeholder`

2. Pass `content` as initial content with `contentType: 'markdown'` in `useEditor` options.

3. In `onUpdate` callback: call `onChange(editor.getMarkdown())` to serialize to markdown.

4. Expose the editor instance via `useImperativeHandle` with a `forwardRef` so EditorPane can call `editor.getMarkdown()` during mode switch. The ref type:
   ```typescript
   export interface WysiwygEditorHandle {
     getMarkdown: () => string;
     setContent: (md: string) => void;
   }
   ```

5. When `content` prop changes AND it differs from the current editor markdown (new file loaded), call `editor.commands.setContent(content, false, { contentType: 'markdown' })`. Use a ref to track the last-set content to avoid re-parse loops.

6. Render: `<EditorContent editor={editor} className="prose prose-orange max-w-none px-6 py-4" />`

7. Wrap the EditorContent in a `<div className="h-full overflow-y-auto">`.

8. Add CSS for hover affordance: target `.ProseMirror > *:hover` with a left-border (2px solid orange-200, transition 150ms). Add this as a `<style>` tag or inline in the component file using a Tailwind `@layer` or plain CSS in a style block.

9. Render `<BubbleToolbar editor={editor} />` (import from ./BubbleToolbar — created in Task 2).

Update the test stubs in `client/src/__tests__/WysiwygEditor.test.tsx` to use real assertions against the rendered component. Tests must pass (GREEN).
