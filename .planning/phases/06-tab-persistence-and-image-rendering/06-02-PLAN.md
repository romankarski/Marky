  <name>Task 1: useTabPersistence and useScrollPersist hooks</name>
  <files>client/src/hooks/useTabPersistence.ts, client/src/hooks/useScrollPersist.ts</files>
  <behavior>
    useTabPersistence exports:
    - loadPersistedTabs(): Array<{path: string; label: string}> — reads 'marky:tabs', returns [] on missing/corrupt
    - saveTabState(tabs: Tab[], activeTabId: string | null): void — writes {tabs: [{path,label}], activeTabPath} to 'marky:tabs'
    - updateRecentFiles(path: string): void — prepend path to 'marky:recent', deduplicate, cap at 5
    - getRecentFiles(): string[] — reads 'marky:recent', returns [] on missing/corrupt

    useScrollPersist exports:
    - useScrollPersist(filePath: string, content: string | null): React.RefObject<HTMLDivElement> — returns a containerRef. Attaches debounced (200ms) scroll listener. On scroll, saves containerRef.current.scrollTop to 'marky:scroll'[filePath]. Uses useLayoutEffect to restore scrollTop when content transitions null → string. Returns the ref to attach to the scroll container element.

    Debounce pattern: useRef storing setTimeout ID, clearTimeout on each new scroll event (same shape as useAutoSave hook in client/src/hooks/useAutoSave.ts — read that file first).

    Scroll restore guard: useRef<string | null>(null) tracking previousContent. Only restore when previousContent is null and content becomes non-null. After restore, set previousContent = content.

    IMPORTANT — Pitfall 3 (stale UUIDs): saveTabState must persist path as the active tab identifier, not the UUID. loadPersistedTabs returns [{path, label}] only. The caller (App.tsx) is responsible for matching the active path back to a fresh UUID after dispatching OPEN actions.

    IMPORTANT — Pitfall 1 (scroll restore timing): If scrollTop reads as 0 after assignment (container not yet scrollable), wrap the assignment in a single requestAnimationFrame call. Log a warning in dev mode if the fallback fires.
  </behavior>
