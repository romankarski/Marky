The existing tree auto-reveal useEffect (around line 252) only fires on `query` transitions (search clear). Add a separate useEffect that calls `expandFolder` whenever the active file changes — covering tab switches, file opens from search results, and split-view pane focus changes.

Add this useEffect after the existing search-clear reveal effect:

```typescript
// Tree reveal: always expand ancestors of the active file whenever it changes
// Covers: tab switch, file open, split-view pane focus change
useEffect(() => {
  const path = activeFocusedTab?.path;
  if (path) expandFolder(path);
}, [activeFocusedTab?.path]); // eslint-disable-line react-hooks/exhaustive-deps
```

This replaces the need to manually track query transitions for reveal — the tree will always stay in sync with the open file. The existing search-clear reveal effect (prevQueryRef) can remain as a safety net but this new effect covers the primary case.

Note: `activeFocusedTab` is derived in Task 1 — this task depends on Task 1 being applied first.
