  <action>
The current onClick handler delegates scroll to the parent (`onHeadingClick`) or calls `scrollIntoView` directly but never updates `activeId`. This means the orange highlight stays on whatever heading was last visible to the IntersectionObserver — which may be the wrong one after a programmatic scroll.

Fix: call `setActiveId(h.id)` immediately at the start of the onClick handler, before delegating the scroll. The IntersectionObserver will continue to handle scroll-based updates; the immediate set just ensures a click always visually confirms immediately.

Change the onClick from:
```tsx
onClick={(e) => {
  e.preventDefault();
  if (onHeadingClick) {
    onHeadingClick(h.id);
  } else {
    document.getElementById(h.id)?.scrollIntoView({ behavior: 'smooth' });
  }
}}
```

To:
```tsx
onClick={(e) => {
  e.preventDefault();
  setActiveId(h.id);  // optimistic update — highlight immediately on click
  if (onHeadingClick) {
    onHeadingClick(h.id);
  } else {
    document.getElementById(h.id)?.scrollIntoView({ behavior: 'smooth' });
  }
}}
```

Also reset `activeId` to null when content changes so a stale highlight from a previous file doesn't persist on the new file's TOC. In the useEffect that runs `[content]`, add `setActiveId(null)` at the top before reconnecting the observer:

```typescript
useEffect(() => {
  setActiveId(null); // reset highlight when file changes
  if (observerRef.current) observerRef.current.disconnect();
  // ... rest of observer setup unchanged
}, [content]);
```
