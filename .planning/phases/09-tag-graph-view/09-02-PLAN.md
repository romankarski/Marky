Edit `server/src/lib/search.ts` to export `TagGraphNodePayload`, `TagGraphLinkPayload`, and `TagGraphPayload`, then add a new `getTagGraphPayload(): TagGraphPayload` method on `SearchService`.

Implementation rules:
- Build `nodes` from `this.docs.values()` only. Do not re-read the filesystem.
- Use `path` as the stable node `id` so client-side persistence can key by path without translation.
- Set `label` to the markdown filename without `.md`.
- Include every indexed markdown file in `nodes`, even when `tags.length === 0`.
- Build links by walking tag buckets and accumulating pairs. If two files share multiple tags, emit one link with `weight` equal to the shared-tag count and `sharedTags` listing those tags.
- Sort nodes by `path` and sort `sharedTags` arrays so the route payload is deterministic for tests.

Do not add a separate `TagGraphService`. Reuse the existing `SearchService` map and tag metadata per the research recommendation.
